# Database Schema

AgentRadar uses Supabase Postgres. All tables live in the `public` schema with Row Level Security enabled.

Migration file: `supabase/migrations/001_initial_schema.sql`

---

## Tables

### `items`

The primary table. Every piece of content ingested from GitHub, Hacker News, or an RSS feed becomes one row.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | `gen_random_uuid()` |
| `title` | text NOT NULL | |
| `url` | text NOT NULL | Original source URL |
| `canonical_url` | text UNIQUE | Normalised URL used for deduplication; nulls don't conflict |
| `source` | text NOT NULL | `'github'` \| `'hackernews'` \| `'rss'` |
| `source_id` | text | Repo full name, HN story ID, or RSS guid |
| `author` | text | |
| `description` | text | Short blurb from the source |
| `raw_content` | text | Full body / README if fetched |
| `raw_data` | jsonb | Verbatim API response payload |
| `published_at` | timestamptz | When the content was published at the source |
| `discovered_at` | timestamptz | When AgentRadar first saw it (default `now()`) |
| `github_stars` | int | GitHub only |
| `github_forks` | int | GitHub only |
| `github_language` | text | GitHub only |
| `hn_points` | int | Hacker News only |
| `hn_comments` | int | Hacker News only |
| `ai_summary` | text | 1‚Äì2 sentence plain-English summary |
| `ai_why_it_matters` | text | Why a GenAI developer should care |
| `ai_category` | text | e.g. `agent-framework`, `rag`, `tooling` |
| `ai_tags` | text[] | Up to 5 lowercase tags |
| `ai_audience` | text[] | e.g. `['ml-engineer', 'backend-dev']` |
| `ai_maturity` | text | `experimental` \| `beta` \| `production-ready` |
| `ai_relevance_score` | numeric(4,3) | 0.000‚Äì1.000; how relevant to GenAI developers |
| `ranking_score` | numeric(10,4) | Computed composite score; default 0 |
| `status` | text | `'new'` ‚Üí `'enriched'` or `'failed'` |
| `error_message` | text | Set when `status = 'failed'` |
| `created_at` | timestamptz | Row creation time |
| `updated_at` | timestamptz | Auto-updated by trigger on every write |

**Indexes:** `source`, `status`, `ai_category`, `published_at DESC`, `ranking_score DESC`, `created_at DESC`, `canonical_url` (via unique constraint)

### `digests`

A periodic (weekly) curated digest generated by the AI pipeline.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | |
| `title` | text NOT NULL | e.g. `"AgentRadar Weekly ‚Äî Jan 6‚Äì12"` |
| `period_start` | date NOT NULL | Monday of the digest week |
| `period_end` | date NOT NULL | Sunday of the digest week |
| `summary` | text | AI-generated intro paragraph |
| `created_at` | timestamptz | |

**Indexes:** `period_start DESC`, `period_end DESC`

---

### `digest_items`

Ordered join table linking a digest to its featured items.

| Column | Type | Notes |
|---|---|---|
| `digest_id` | uuid FK ‚Üí `digests(id)` | CASCADE delete |
| `item_id` | uuid FK ‚Üí `items(id)` | CASCADE delete |
| `rank` | int NOT NULL | 1 = top item |
| `section` | text | Optional grouping label |

**Primary key:** `(digest_id, item_id)`

---

### `rss_feeds`

Configuration table for RSS sources. Seeded by the migration; add new feeds here without a code deploy.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | |
| `name` | text NOT NULL | Human-readable label |
| `url` | text NOT NULL UNIQUE | Feed URL |
| `active` | boolean | Set to `false` to pause ingestion |
| `category_hint` | text | Passed to AI enrichment as context |
| `created_at` | timestamptz | |

**Seed rows (6):** OpenAI Blog, Anthropic News, GitHub Blog, LangChain Blog, Vercel Blog, Simon Willison's Blog

## Security

- **RLS enabled** on all four tables.
- `anon` role has **SELECT only** ‚Äî no INSERT, UPDATE, or DELETE.
- All writes go through the **service role key** (`SUPABASE_SERVICE_ROLE_KEY`), used exclusively in server-side scripts and API routes ‚Äî never in client components.
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` is safe for the browser; it is subject to RLS.

## Running the Migration

1. Open [Supabase Dashboard ‚Üí SQL Editor](https://supabase.com/dashboard/project/imtslobvotyythfxztmj/sql/new)
2. Click **New query**
3. Paste the contents of `supabase/migrations/001_initial_schema.sql`
4. Click **Run** (or `Ctrl+Enter`)
5. Verify in **Table Editor** that `items`, `digests`, `digest_items`, and `rss_feeds` appear
6. Check **Authentication ‚Üí Policies** to confirm RLS policies are active

## TypeScript Types

All types are defined in `lib/db/types.ts` and re-exported from `types/index.ts`.

Key shapes:

| Type | Use |
|---|---|
| `Item` | Full row read from DB |
| `ItemInsert` | Minimal shape for ingestion upserts |
| `ItemEnrichmentUpdate` | Fields written by the AI pipeline |
| `ItemFailureUpdate` | Fields written on enrichment error |
| `ItemCard` | Lightweight shape for feed/search UI |
| `Digest` / `DigestInsert` | Digest row shapes |
| `DigestItem` / `DigestItemInsert` | Join table shapes |
| `DigestWithItems` | Digest + joined items for detail page |
| `RssFeed` / `RssFeedInsert` | RSS feed config shapes |
